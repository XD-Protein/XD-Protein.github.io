南邮ctf的部分re

1.Py交易

根据题目知道这是关于python的逆向题，百度找到一个python的反编译工具: Easy Python Decompiler ;工具下载地址：http://sourceforge.net/projects/easypythondecompiler/?source=directory 安装好之后，直接将题目的pyc文件拖过去打开，得到代码

过程很简单，是将flag每个字符的ascii吗与32异或，然后加16，然后base64加密。

所以将correct解一次base64，然后每个字符ascii码都减下16，接着与32异或即可得到flag

2.Wxyvm

1.分析main()

elf文件，用64ida打开，查看main()的源代码:

要求满足，长度为24，flag经过sub_4005B6函数的变换后，要与dword_601060相等

2.分析sub_4005B6()

byte_6010C0中有15000个元素，每三个分为一组。每组的第一个元素确定进行哪种运算(case v0)，第二个元素(result)确定flag的哪一位进行运算，第三个元素(v3)与flag对应的那一位进行运算。

3.运行逆向脚本

因为15000个元素实在是太多了，选中再复制太不方便了,就在想怎么能得到这些数据。

百度得知 ida的File->Script file 可以直接运行脚本，有两种: idc 和 python. 这里使用python

原函数是从0-15000依次对flag进行操作的，所以逆向需要从15000倒过来进行逆操作。运行脚本即可得到flag

这道题最大的收获是知道了ida有一个专门的idapython插件，可以用python来写ida的脚本，在ida里运行，这样可以直接得到数据，更加方便。

3.maze

1.分析开头

elf 用ida64打开，f5看main()的源码。开头是这样的

这是说flag为24长度，并且要以“nctf{”开头，“}”结尾。 然后从第六位开始进行一些操作。

结合流程图

百度得知有很多ida常用宏定义。 SHIDWORD为*(&v9+1) ，即取v9的下一字节 。

2.分析四种操作

‘o’ ‘O’对v9的下一位进行操作，‘0‘和’.‘对v9进行操作。

每一次循环都会进行这样的判断

3.走迷宫

进入sub_400690 ,看见这样一句话:result = (_BYTE )(a1 + a2 + 8LL * a3); a1,2,3对应三个参数。

所以可以推出 v9的下一字节 表示列，v9表示行， O为左移一位，o为右移一位，.为上移一位，0为下移一位

asc_601060为迷宫，每行八个元素。

根据这句话知道以’#’结尾,所以走的顺序为 ：右下右右下下左下下下右右右右上上左左

对应字符 o0oo00O000oooo..OO 加上开始头的限制就得到了flag。
